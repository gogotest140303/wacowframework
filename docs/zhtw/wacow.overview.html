1.1 簡介

Wacow Framework (以下簡稱 WF) 是基於 Zend Framework 上，一個用來快速產生擁有後台管理的 CMS 框架；其全名為 WAbow Constructing Outstanding Web application 。 WF 提供多數專案常用的功能，以及結合經驗與時下流行技術的半自動化管理平台，其主要目的如下：

  • 減少重複的細部程式調整時間，讓開發者將心力放在功能面的調整上。
  • 保留可客製化的空間。
  • 透過 AJAX 技術減少傳統 HTML 連結重載頁面所帶來的效能問題。
  • 線上管控客戶的系統狀況，尤其當開發者無法直接接觸上線系統時。
  • 定義抽象架構，並提供預設並實用的實作方式；如此可提供開發者擴展與客製化的空間。

1.1.1 特色

主要核心

  • 採用 Zend Framework 及其 MVC 架構。
  • 可驗證式標準化表單元件。
  • XHTML + CSS 版型。
  • 完全 WEB 化的 Sitemap 操作機制。
  • 分頁物件。

前端核心

  • 採用 jQuery 函式庫。
  • 後台部份採用 XHTML 加 CSS 排版。

樣版系統

  • 採用 Smarty 。
  • Master Page 模式。
  • 單一區塊快取。
  • 依賴式 JavaScript 載入功能。

後台管理系統核心

  • 基本登入功能。
  • 基本列表、新增、修改及刪除功能。
  • 可客製化的列表頁及編輯頁。
  • 可程式化 Layout 機制。

其他

  • 專案產生器。
  • 模組產生器。
  • 縮圖助手。
  • 權限管理。
  • 檔案上傳助手。

1.1.2 TODO

以下功能要先考慮實作的難易度。

  • 模組壓縮檔上傳自動安裝機制
    • 參考： SMF 的後台外掛安裝機制。
  • 使用者群組權限機制
    • 參考 Zend_Acl
  • 可程式化快取機制
    • 可更換的儲存機制：例如預設為 disk ，必要的話可抽換為 memcached 。 (Zend_Cache)
    • 開發時要能選擇關閉快取機制：例如開發時我們會需要立刻看到後台資料更動的結果以便測試，但是上線後為了效能考量就要改用快取。
  • 線上偵錯機制
    • 開發者模式：例如先連結到某個頁面建立 Debug Session 後，再導向正常的前端畫面。此時前端畫面會出現一個 Debug 用的 DIV 視窗。
    • 測試者模式：同開發者模式，但訊息會送至後台或開發者信箱，而非顯示給使用者觀看。
  • AJAX 後端通訊協定
    • 可後端程式化的 JavaScript 模組：讓 AJAX 行為從 HTML 中分離出來，並在後端 PHP 程式中直接控制，而不用撰寫過多 JavaScript 程式碼。
    • 參考： xajax 、 jQPie 。
    • 參考：另外 PHPe 的 cid73 所發表的「元編程初探: 使用 PHP 作為 Javascript 的元語言」。
    • 加密協定：http://www.coolcode.cn/category/phprpc/
  • 持續化物件服務
    • 參考： JPOX 讓全域物件常駐在記憶體中，避免重複建構的效率問題。
  • 交換式資料更新機制
