A.1. 說明

Wacow 專案的程式碼撰寫標準採用 Zend Framework 的 Coding Standard ，並加入詳細的內部撰寫標準。

A.2. 命名

A.1.1. 類別成員命名

private 屬性或函式名稱前加上底線， protected 和 public 一律不加底線。

A.1.2. 類別方法命名

  • 在某事件之前，範例：
    • beforeEvent
  • 在某事件之後，範例： 
    • afterEvent
  • 是否有某屬性，範例： 
    • hasAttribute
  • 是否為某類型，範例： 
    • isBoolean
  • 是否為某狀態，範例： 
    • isEnabled
