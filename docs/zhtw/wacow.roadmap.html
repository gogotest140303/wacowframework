主要核心

Wacow 主要核心在於包裝 Zend Framework ，使它能夠有建置專案的基本雛形。

目錄結構

Wacow 專案的既定目錄結構如下： 

project/ ---------------------- 專案目錄
        app/ ------------------ 應用程式目錄
            base/ ------------- 基本 Controller 及 Model
                 Controller/
                 Model/
            etc/ -------------- 設定
            lib/ -------------- 第三方類別庫
                PHPMailer/
                Smarty/
                Wacow/
                Zend/
            mod/ -------------- 模組
                default/
                admin/
            sql/ -------------- 存放 SQL 指令
            tmp/ -------------- 暫存資料夾
        debug/ ---------------- 除錯用
        doc/ ------------------ 文件
        pub/ ------------------ 公開
            css/ -------------- CSS
            img/ -------------- 圖片
            js/ --------------- JavaScript
            lib/ -------------- 第三方 JS 庫
            theme/ ------------ 後台佈景主題
        index.php ------------- Bootstrap

Application

提供專案應用程式的目錄管理以及相關物件的取得，例如資料庫轉接器、快取及設定檔物件等。

View 管理員

主要功能有三：

  • 依照是否為 Ajax Request 來建構對應的 View Object 。

  • 動態取得共用樣版。

  • Assign 需要給 JavaScript 的變數。

DataSet

主要為資料庫、 RSS 或其他可被分頁的資料集。作法要考慮以下兩點：

  1. 結合分頁介面並且取得關連欄位，所以在 getIterator 時要能呼叫 callback 來塞入關連欄位。可參考 ocar/seller/index 裡，找出車商對應的車輛的作法。
  2. 可能會搭配 Ajax ，因此就沒辦法在 Web Compnent 裡面做分頁。

樣版語法

自訂一組樣版標籤來隱藏 Smarty 一些較為複雜的語法，並且在裡面直接與 Action Controller 溝通。

動態載入區塊

<wa:block file="/path/to/templatefile" bind="method:test" cachetime="200" />

連結

<wa:link path="/path/to/action">Link Text</wa:link>

彈出視窗

<wa:popup iframe="yes" src="/path/to/some.html" width="200" height="100">Link Text</wa:popup>

表單

<wa:form action="/action" method="post">
<wa:text name="text1" mutiple="yes" /> // 文字欄位
<wa:select name="select1" mutiple="yes" bind="abc" /> // 下拉選單
<wa:editable name="editable1">Text</wa:editable> // 就地編輯
<wa:htmleditor name="html1"></wa:htmleditor> // HTML 編輯器
<wa:downlist name="category1"></wa:downlist> // 多層式動態下拉選單
<wa:downlist name="category2" bind="category1"></wa:downlist>
</wa:form>

動態載入 JavaScript 及 CSS

<wa:js name="jquery.thickbox" required="jquery" /> // 自動抓取 /public/scripts/jquery/thickbox.js
<wa:css name="admin.user" /> // 自動抓取 /public/styles/admin/user.css

資料顯示 (含分頁) 

<wa:datagrid name="datagrid1" bind="cars">
<wa:column name="title">
Custom style of colunm
</wa:column>
<wa:function>
<wa:link path="/add">Add Row</wa:link>
</wa:function>
</wa:datagrid>

分頁介面

<wa:page name="page" type="slide" bind="AnyDataSet" pagesize="config:search:pageSize" layersize="config:search:layerSize" style="digg" />

另外是 Head 的部份，要能夠把動態載入的 JavaScript 和 CSS 變成 HTML 語法。

表單自動化

表單自動化主要目標是在 Controller 建立驗證機制，然後在樣版的地方以樣版語法來連結以提供 Ajax 驗證的效果。

Ajax 驗證系統

前端表單所有欄位，以 Ajax 送到後端驗證。當沒有開啟 JavaScript 時，則改用 FlashMessanger 來代替。

表單元件

主要包含：

  • 文字欄位 (可多行)
  • 密碼欄位
  • 下拉選單
  • 數字範圍選擇器
  • 日曆選擇器
  • 日期區間選擇器
  • 顏色選擇器
  • 檔案上傳

Theme 代換

Theme 包含 CSS 、 JavaScript 及 Image 等，預設為 wacow 。

Theme 要能向樣版註冊會用到的 CSS 和 JavaScript 的能力，其中包含對 IE6 的判斷。

輔助功能

以下的 Action Helper 是常在 Action Controller 裡使用的功能：

檔案上傳助手

搭配檔案上傳的表單元件，可以決定上傳的檔案類型。然後上傳檔案後將這些檔案存放在資料庫中，讓其他類型的資料可以連結利用。

縮圖助手

提供縮圖功能，並且可決定是不是要補白。

版面管理員

可以指定主樣版，並且提供區塊快取功能。

分頁助手

協助開發者在 Action 中建立 Pager ，並且在 View 中。
